# Архитектура

## Обзор

- **backend-public** — сервис с публичным REST/GraphQL API, доступ к агрегаторам котировок и кэшу.
- **packages/shared** — общие типы DTO, ошибки, константы версий API.
- **packages/api-client** — тонкий клиент для потребителей API.
- **apps/web** — демо UI поверх того же API.
- **apps/marketing** — статический маркетинг без доступа к внутренним системам.

## Потоки данных

1. Клиент (браузер, SDK, примеры) обращается к `backend-public`.
2. Backend нормализует символы, применяет лимиты и возвращает типизированные ответы.
3. Кэш (Redis или in-memory на этапе MVP) снижает нагрузку на внешние источники.

Детали сетевых границ и деплоя дополняются по мере реализации.
