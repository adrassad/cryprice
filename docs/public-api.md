# Публичное API

Черновик контракта. Конечные пути, схемы и версионирование фиксируются при реализации `backend-public`.

## Базовый URL

- Production: `TBD`
- Локально: см. `CRYPRICE_PUBLIC_API_URL` в `.env.example`

## Версионирование

Рекомендуется префикс `/v1/` для стабильных маршрутов.

## Аутентификация

- Публичные read-only эндпоинты могут работать без ключа.
- При введении квот — заголовок `Authorization: Bearer <token>` или query-параметр по спецификации.

## Примеры эндпоинтов (план)

| Метод | Путь | Описание |
|-------|------|----------|
| GET | `/v1/health` | Проверка доступности |
| GET | `/v1/prices` | Список цен по символам |

Тела запросов/ответов и коды ошибок описываются в OpenAPI по готовности backend.
